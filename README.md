Конфигурационный инструмент для учебного языка (Вариант 7)

Описание проекта

Данный проект представляет собой реализацию инструмента командной строки (CLI) для учебного конфигурационного языка по варианту №7. Программа преобразует текст на специальном конфигурационном языке в формат YAML с проверкой синтаксиса и поддержкой констант на этапе трансляции.

Функциональные требования

Аргументы командной строки:

Входной текст на учебном конфигурационном языке принимается из стандартного ввода (stdin).
Выходной текст на языке YAML попадает в стандартный вывод (stdout).
Синтаксические ошибки выявляются с выдачей понятных сообщений.
Поддерживаемый синтаксис (Вариант №7):

Многострочные комментарии:

text
--[[
Это многострочный
комментарий
]]
Числа: Регулярное выражение -?(\d+|\d+\.\d*|\.\d+)([eE][-+]?\d+)? (целые, дробные, с плавающей точкой в научной нотации).
Массивы: ( значение, значение, значение, ... )
Словари (объекты):

text
{
    имя : значение,
    имя : значение,
    ...
}
Имена: Регулярное выражение [_a-zA-Z]+.
Значения: Числа, Массивы, Словари.
Объявление константы: имя = значение;
Использование константы: $имя$ (результатом является значение константы).
Тестирование:

Все конструкции языка (с учётом возможной вложенности) должны быть покрыты тестами.
Необходимо предоставить 2 примера описания конфигураций из разных предметных областей (например, настройки сервера и конфигурация игры).
Требования к реализации:

Реализация выполняется на языке программирования по выбору студента (Python, Java, Go, Rust, C++ и др.).
Обязательно использование специализированного инструмента синтаксического разбора (парсера), такого как ANTLR, Lark, yacc/bison, Peggy и т.п. Решения без парсера не засчитываются.
